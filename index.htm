KeyCore

A USB bootable keyboard tester

Download KeyCore (v1.0) View on GitHub

What is it?

KeyCore is a lightweight system (weighing in at just 64 megabytes!) built with Tiny Core Linux and Python for the purpose of testing keyboards.

It was originally designed for HandyGeek to test laptop keyboards being stripped for spare parts; a lot of which didn't have storage devices with an operating system.

Installing and booting from a USB pen drive

Start by downloading the latest version of the KeyCore ISO using the download link at the top of this page.

Once you have downloaded the ISO, you will need to find a USB pen drive with at least 64 megabytes of space and software capable of creating bootable USB drives. If you are using Windows, I highly recommend using Rufus.

If using Rufus to create your bootable USB drive, simply choose the USB drive from the device dropdown list, select the ISO file you downloaded and then hit the start button; it's that simple!

Once you have finished setting up your USB drive, simply plug it into a computer that supports USB booting, power up and within roughly 10-15 seconds you'll be in the KeyCore environment, which will display the key code of the key you last pressed.

Limitations

Currently, KeyCore is unable to detect modifier keys (CTRL, ALT, shift, caps lock etc.) being pressed individually. In order to test these keys, you must use them in conjunction with a key they will modify. For example, pressing ALT + A will give you a different key number than pressing A on its own, likewise pressing the A key whilst having caps lock enabled will send a different signal than if caps lock was disabled.
